

#  Maven Market Lakehouse

[![CI/CD](https://github.com/maven_market_admin/maven-market-lakehouse/actions/workflows/ci-cd.yml/badge.svg)](https://github.com/YOUR_USERNAME/maven-market-lakehouse/actions/workflows/ci-cd.yml)

A production-grade **Azure Databricks Lakehouse** implementing an end-to-end **Medallion Architecture (Bronze → Silver → Gold)** with real-time + batch ingestion, SCD Type 2 tracking, data quality enforcement, and Unity Catalog governance.

---

##  Quick Overview

*  Multi-source ingestion: **ADLS (CSV), MongoDB (Fivetran), Kafka (Streaming)**
*  Medallion architecture using **Lakeflow Declarative Pipelines (DLT)**
*  SCD Type 2 for customers, products, stores
*  Data quality + quarantine framework
*  Row-Level Security (RLS) + Column-Level Security (CLS)
*  Star schema + business-ready aggregates
*  CI/CD using Databricks Asset Bundles + GitHub Actions

---

##  Architecture Overview

```
ADLS (CSV)     MongoDB (Fivetran)     Kafka (Streaming)
     │                  │                  │
     └──────────► Bronze Layer ◄──────────┘
                      │
                      ▼
                Silver Layer
     (cleaning, SCD, validation, quarantine)
                      │
                      ▼
                Gold Layer
     (dimensions, facts, aggregates)
                      │
                      ▼
               Governance Layer
         (RLS, CLS, GRANTS)
```

##  Medallion Architecture

###  Bronze Layer

* Raw ingestion (no transformation)
* Metadata enrichment (timestamps, source tracking)

###  Silver Layer

* Type casting and date parsing
* Deduplication (with watermark for streaming)
* JSON flattening (`from_json`)
* SCD Type 2 (`apply_changes`)
* Data validation
* **Valid + Quarantine split**

###  Gold Layer

#### Dimensions

* `dim_customer`
* `dim_product`
* `dim_store`

#### Facts

* `fact_sales` → revenue, cost, profit
* `fact_returns` → return quantities

#### Aggregates

* `agg_daily_sales`
* `agg_monthly_sales`
* `agg_store_performance`
* `agg_return_kpi`

##  Data Quality & Quarantine

### DLT Expectations

```python
@dlt.expect("valid_quantity", "quantity > 0")
```

### Quarantine Pattern

Each table produces:

* `*_valid`
* `*_quarantine`

With:

* `_quarantine_reason`
* `_quarantine_timestamp`

---

##  Data Governance (Unity Catalog)

### Row-Level Security (RLS)

* Based on user groups (`is_member()`)
* Region-based filtering

### Column-Level Security (CLS)

* Financial fields masked:

  * `total_revenue`
  * `total_cost`
  * `total_profit`
* PII masked:

  * `customer_name`

### Secure Views

* `v_secure_fact_sales`
* `v_secure_fact_returns`
* `v_secure_dim_customer`
* `v_secure_dim_product`
